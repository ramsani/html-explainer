# Roadmap

This roadmap is intentionally small. The project should grow by improving trust, examples, and behavior verification before adding heavy features.

## Next

- Keep scenario-based behavior tests current as patterns evolve.
- Add more before/after demos only for workflows where HTML removes real mental work.
- Improve the example gallery with screenshots and decision-first previews.
- Strengthen semantic checks when a weak artifact can still pass.

## Later

- Add richer artifact templates after scenario tests protect routing quality.
- Add optional installer presets for different Claude Code setups.
- Add community examples and contribution guidance for new patterns.
- Explore a local read-only artifact memory connector after the file model is stable.

## Current Principle

Do not expand the system until the current behavior is easier to verify.

The priority is not more HTML. The priority is decision-ready HTML that reliably helps users inspect evidence, understand risk, choose the next action, and continue work with less ambiguity.
