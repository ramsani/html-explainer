# Docs Coherent Goal

Use when changing README, docs, commands, or contribution guidance.

```text
/goal docs are coherent when README explains the user value in under two minutes, DOCS_MAP tells agents what to read first, CONTRIBUTING and CI require the same deterministic checks, claims do not exceed implemented behavior, and /goal contracts cover quality judgment that scripts should not pretend to evaluate. Do not finish if docs contradict the installer, CI, command behavior, or local memory implementation.
```
