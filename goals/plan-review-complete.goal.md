# Plan Review Complete Goal

Use with `/check-the-plan`.

```text
/goal the plan review is complete when the user can see what the plan tries to achieve, what repo evidence supports or contradicts it, which assumptions matter, what can break, what must be verified before implementation, and whether to approve, revise, reject, split, or ask for a sharper plan. Do not finish if the review approves without evidence, hides uncertainty, or leaves the user to invent the next command.
```
