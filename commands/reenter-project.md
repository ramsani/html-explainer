# /reenter-project

Create a repo/project recap that helps the user re-enter context quickly.

## Process

1. Apply `docs/DECISION_GATE.md`.
2. Use pattern 17 unless the task is clearly narrower.
3. Inspect README, changelog, entry points, scripts, CI, commands, patterns, and recent commits.
4. Build the evidence sheet with `docs/FACT_SHEET.md`.
5. Generate the smallest useful project recap.
6. Apply `docs/STYLE.md`.
7. Apply `docs/QUALITY_BAR.md`.
8. Apply `docs/CHAIN.md` so the next repo action is obvious.
9. Apply `docs/DELIVERY.md`.

## Must Answer

- What is this project?
- What value does it provide?
- How does work flow through it?
- What is solid?
- What is risky or unclear?
- What is the next highest-leverage action?

## Output

End with one recommended next prompt and useful alternatives for the next likely step.
