# Human Interface

Use this when writing visible artifact copy, buttons, choices, and next-step invitations.

The interface should feel natural, respectful, and powerful because it is simple.

## Rule

Invite, do not push.

The user should always feel:

- I know what this is for.
- I know what was checked.
- I know what can go wrong.
- I know the best next move.
- I can choose, ignore, save, continue, or ask for deeper thinking.

## Quiet Invitations

Use soft, optional language:

```text
Want to think this through?
Compare options
Review before acting
Continue from here
Save for later
Open the saved version
```

Avoid pushy language:

```text
You should now...
You must...
Required next step
Click here to proceed
```

## Minimal Choice Set

Most artifacts should expose 2-4 choices, not a menu of everything.

Good defaults:

```text
Use this
Revise
Compare
Think this through
Save for later
```

The best next move can be recommended, but the user must still see that they are free to choose another path.

## Power In The Mouse

When interaction exists, it must give the user real control:

- compare;
- filter;
- tune;
- preview;
- copy;
- export;
- save;
- continue.

Do not add controls that only decorate the page.

## Tone

Be calm, capable, and plain.

Use short labels. Explain only what helps action.

The artifact should feel like a helpful expert standing beside the user, not like a system showing its internal process.

## Pattern Requirement

Every pattern must make the user's agency visible:

- what choice the user can make;
- what action is safest;
- what optional deeper function could help, such as thinking through, comparing, reviewing, saving, or continuing;
- how to leave without losing context.
