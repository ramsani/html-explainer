# Memory Access

Use this when an artifact may become future context.

The user should feel that useful work is organized and easy to find, without seeing archive mechanics.

## Rule

Give a tiny memory receipt.

Do not explain indexing, metadata, schemas, folders, or lifecycle rules unless the user asks.

## Visible Copy

Use one of these labels:

```text
Saved for later
Find it fast
Not saved
```

## When Saved

Include this compact block near the end of the artifact:

```text
Saved for later
Where: <plain collection or topic name>
Find it fast: ask for "<short natural-language retrieval phrase>"
```

Keep it to 1-3 lines.

Good retrieval phrases:

```text
open my saved decision about pricing
show the last repo review
find the plan for onboarding
continue from the saved artifact about the installer
```

## When Not Saved

Use a quiet line:

```text
Not saved: this was only useful for this example.
```

Do not add access instructions for low-value scratch work.

## Final Chat Response

If the artifact was saved or should be saved, include one short line:

```text
Saved for later: <where/how to find it>
```

If it should not be saved, include:

```text
Not saved: <short reason>
```

## Hard Limits

- Do not turn every answer into a memory report.
- Do not expose internal metadata names as the main user-facing value.
- Do not save private or stale material without saying so.
- Do not imply automatic search exists when only a local folder or manual retrieval exists.
