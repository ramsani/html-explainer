# Insight Surface Loop

`Insight Surface Loop` is the thinking method behind `/think-with-me-about`.

It helps users move from unclear thinking to actionable insight with a decision-ready HTML surface.

The method should feel simple to the user and rigorous underneath.

## Core Promise

```text
unclear idea -> visual thinking surface -> evidence and assumptions -> tension and inversion -> provisional insight -> action -> next prompt -> reusable local knowledge
```

The goal is not to create a beautiful reflection page.

The goal is to help the user think better, faster, and with less ambiguity.

## When To Use

Use this method when the user wants to think through:

- an unclear idea;
- a strategic decision;
- an opportunity;
- a complex problem;
- a product direction;
- a repo or project future;
- a hard trade-off;
- a personal or business hypothesis;
- a plan that needs sharper reasoning.

Do not use it for:

- simple factual answers;
- single commands;
- tiny edits;
- low-consequence notes;
- tasks where Markdown is clearly enough.

## Loop

```text
Frame -> Expand -> Ground -> Stress -> Synthesize -> Decide -> Preserve -> Re-enter
```

## 1. Frame

Clarify what the user is really trying to think about.

Required outputs:

- topic;
- real question;
- desired decision or insight;
- context;
- constraints;
- why this matters now.

The artifact should make the real thinking target visible above the fold.

## 2. Expand

Open the thinking space before narrowing too early.

Useful lenses:

- perspectives;
- stakeholders;
- alternatives;
- hidden options;
- adjacent problems;
- opportunity angles;
- second-order consequences.

The goal is not more information. The goal is better option space.

## 3. Ground

Separate what is known from what is assumed.

Required categories:

- facts;
- evidence;
- inferences;
- assumptions;
- hypotheses;
- unknowns;
- constraints.

Use honest placeholders when needed:

```text
unknown
not inspected
not verified
not enough evidence
stale risk
requires refresh
source missing
```

Do not invent facts, metrics, sources, or confidence.

## 4. Stress

Attack the idea before trusting it.

Required moves:

- inversion: how could this fail?
- pre-mortem: what would make this look foolish later?
- contradiction check;
- weakest assumption;
- missing evidence;
- downside and opportunity cost.

A thinking surface without stress testing is not strong enough.

## 5. Synthesize

Reduce noise into insight.

Required outputs:

- emerging pattern;
- strongest insight;
- provisional conclusion;
- confidence level;
- what would change the conclusion.

If no conclusion is justified, say so and explain what evidence is missing.

## 6. Decide

Convert thinking into action.

Required outputs:

- recommended next action;
- smallest useful experiment;
- what to stop doing;
- what to investigate next;
- decision status: decide now / investigate / defer / reject / reframe.

The loop must not end as open-ended reflection.

## 7. Preserve

Decide whether the artifact should enter local artifact memory.

Required outputs:

- archive recommendation: save / do not save;
- suggested validity: temporal / replaceable / evergreen / private / do-not-reuse;
- freshness risk;
- suggested title;
- suggested tags;
- why preserve or discard.

This should align with `docs/ARTIFACT_MEMORY.md` when present.

## 8. Re-enter

Prepare the next thinking cycle.

Required outputs:

- next thinking prompt;
- focused follow-up question;
- evidence needed before next cycle;
- block conditions.

The next prompt should preserve intent, evidence basis, assumptions, risks, conclusion, and unknowns.

## Quiet Thinking Models

The user should not have to choose a model manually.

Use models quietly when they help:

- Socratic questioning;
- inversion;
- pre-mortem;
- first principles;
- second-order thinking;
- Feynman technique;
- ladder of inference;
- Cynefin-style problem typing;
- OODA-style iteration;
- Double Diamond divergence/convergence;
- trade-off analysis;
- decision matrix;
- opportunity cost;
- systems thinking;
- constraint analysis.

Show the effect of the model, not an academic explanation of the model.

## HTML Surface Structure

A strong artifact includes these zones.

### North Star

- topic;
- real question;
- desired decision or insight;
- provisional conclusion;
- confidence;
- next action.

### Evidence Board

- facts;
- evidence;
- constraints;
- unknowns;
- freshness risk.

### Perspective Map

- perspectives;
- alternatives;
- stakeholders;
- hidden options;
- second-order effects.

### Tension Map

- contradictions;
- trade-offs;
- risks;
- weakest assumptions;
- what could change the conclusion.

### Inversion Panel

- how this fails;
- what to avoid;
- what would make the idea fragile;
- what would make the conclusion wrong.

### Insight Card

- strongest insight;
- provisional conclusion;
- confidence;
- why it matters;
- what changed in the user's understanding.

### Action Queue

- do now;
- test next;
- investigate;
- stop doing;
- defer or discard.

### Preserve / Re-enter

- archive recommendation;
- suggested artifact memory class;
- suggested tags;
- next prompt;
- follow-up question.

## P0 Gates

Do not deliver a serious Insight Surface artifact unless it passes these gates:

- The topic and real question are clear.
- There is a provisional conclusion or a clear reason why one is not justified.
- Facts, inferences, assumptions, hypotheses, and unknowns are separated.
- At least one critical tension or risk is visible.
- Inversion or stress testing is included.
- There is a concrete next action.
- There is a copy-ready next prompt.
- There is an archive recommendation.
- No fake certainty, fake metrics, or unsupported claims are present.
- HTML adds value over Markdown.

## Anti-Slop Rules

The artifact must not:

- pretend certainty without evidence;
- invent facts, sources, metrics, or confidence;
- produce generic insight like "consider the pros and cons";
- hide assumptions;
- skip inversion;
- end without action;
- end without a next prompt;
- show complex visuals that do not change the thinking;
- save low-value thinking into artifact memory by default;
- use HTML if Markdown is clearly enough.

## Success Definition

The method succeeds when the user feels the artifact helped them think, not merely read.

The output should make the next better thought easier.
