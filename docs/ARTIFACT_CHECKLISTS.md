# Artifact Checklists

Checklists are the quality floor for serious artifacts.

## P0 Blocks Delivery

- Primary intent is visible before the user must search.
- Evidence basis is visible.
- Facts, inferences, assumptions, and unknowns are separated.
- Risk or known limits are visible.
- Recommendation or next action is visible.
- HTML advantage is stated.
- Next prompt or next command is copy-ready.
- No fake metrics, fake certainty, or unsupported current-status claims.

## P1 Required For Serious Work

- Secondary intents are covered when they affect the decision.
- Missing checks are visible near the recommendation.
- The artifact budget is justified.
- Archive recommendation exists when the artifact may be reused.
- Mobile/narrow reading remains usable.

## P2 Polish

- Copy buttons for prompts or export blocks.
- Filters or toggles only when they reduce mental work.
- Compact legend for confidence, severity, or validity.
- Small screenshot or preview when it helps public trust.

## Pattern Package Check

A pattern package should include:

- pattern file;
- example artifact or template;
- evidence requirements;
- quality checklist;
- failure modes;
- validation hook when structure can be checked without an LLM.
