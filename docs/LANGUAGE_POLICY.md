# Language Policy

The repository language is English.

The user-facing experience should match the user's language.

## Rules

- Keep repository docs, command files, pattern files, scripts, examples metadata, and GitHub-facing material in English.
- Respond to the user in the language they use unless they ask for another language.
- Generate HTML artifact copy in the user's language unless the artifact is meant for a different audience.
- Keep code identifiers, filenames, schema keys, and validation fields in English.
- Keep internal instructions simple and operational. Do not expose theory-heavy language to the user.

## Why This Matters

English keeps the repo easier to install, share, search, validate, and contribute to.

Language matching keeps the user experience low-friction. A Spanish-speaking user should receive Spanish guidance and Spanish artifact text unless they ask otherwise.

## Artifact Rule

Before delivering an artifact, decide:

```text
User language:
Artifact audience:
Artifact language:
Reason:
```

If the user language and artifact audience differ, use the audience language and state it briefly.

## Good Behavior

- User writes in Spanish about a repo decision: respond in Spanish and create the artifact in Spanish.
- User writes in English about a public README: respond in English and create the artifact in English.
- User writes in Spanish but asks for an English sales page: respond in Spanish, create the sales page in English, and explain that choice in one line.
