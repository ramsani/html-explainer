5 elementos fuera — rechazados con razón

Slide deck mode
html-explainer es para decisiones, no presentaciones. Viewport-snap es medio diferente. visual-explainer tiene su propio nicho.
Vercel sharing (share.sh)
Dependencia externa, fuera del scope. html-explainer tiene memoria local propia.
surf-cli image generation
Depende de herramienta externa. Genera decoration, no decision support.
Chart.js
Library dependency externa. SVG inline con sparklines es suficiente.
anime.js
Overkill. CSS transitions con stagger son suficientes. Animación compleja no es el foco.

Qué podría salir mal

Riesgo bajo
CSS_PATTERNS.md muy extenso
Que el LLM lo ignore por las ~400 líneas.
Si pasa: resumir a las 10 secciones más críticas.
Riesgo bajo
Conflicto con STYLE.md existente
Agregar a un archivo que ya tiene contenido puede causar conflicto de formato.
Si pasa: git restore para revertir.
Riesgo medio
Font pairings usados en todo
Si el LLM los usa indiscriminadamente, artefactos se vuelven más lentos y verbosos.
Agregar nota: solo cuando el patrón lo requiera (05, 21). Default sigue siendo system fonts.
Riesgo bajo
ANTI_SLOP crece fuera de scope
Que la sección visual confunda con la de decisión ya existente.
Si pasa: separar en dos archivos si es necesario.

Si algo sale mal

Paso 1 — Revertir todo
Si los cambios rompen validación, revertí todo inmediatamente.

Paso 2 — Borrar archivo nuevo

Después de rollback, correr validaciones.
bash scripts/validate-patterns.sh && bash scripts/validate-goals.sh
Paso 4 — No forzar
Si algo no pasa validación después de rollback, no usar force. Preguntar.
