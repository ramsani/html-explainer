Pattern 21 — Visual Direction Explorer

Empty State Direction Explorer — Example

Compare live alternatives for a no-tasks state before choosing UI tone.

Intent distillation

Primary intent: choose the best visual direction for an empty state.

Secondary intents covered: compare tone, density, primary action clarity, state fit, and surface resilience.

Base concept: show real alternatives so the user chooses by seeing, not imagining.

Artifact budget

HTML advantage: live rendered alternatives beat prose descriptions.

Budget: standard.

HTML benefit: shows tone, density, and action clarity side by side.

Mental work removed: imagining how each direction would feel in the product.

Evidence inspected

Product context, empty-state goal, primary action, state requirements, and light/dark surface needs.

A — Minimal

No tasks yet

When you create a task it will show up here.

Calm, low density, best when surrounding UI already has personality.

B — Illustrated
List → Tasks

Start your first list

Group related work and watch progress roll up automatically.

Explains the object model without a long onboarding paragraph.

C — Playful
Nothing here

Nothing on your plate

Enjoy the quiet, or add something to get moving.

Adds warmth. Use only if product tone can support gentle motion.

D — Instructional

Set up this project

  1. Create your first task.
  2. Add a due date.
  3. Invite a teammate.

Highest density. Best for first-run onboarding, not repeat empty views.

Recommendation

Choose B — Illustrated for a new product surface because it explains the list/task model while keeping the action clear. Use A — Minimal once users already understand the product.

Next action: test B and A in the real surrounding layout before implementing illustration polish.
