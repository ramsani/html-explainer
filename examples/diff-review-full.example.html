Finding 1 — Copy overstates the data

The card title says Important action, but the implementation sorts by dueDate, not by priority. This can mislead users.

Fix: rename to Next action unless real priority data is used.

Finding 2 — Missing clear/reset test

Tests cover empty state and normal rendering, but not the behavior when the task list changes after completion or clearing.

Fix: add a test for changing from populated to empty list.

Finding 3 — Layout risk is low but visual check is needed

No global layout files changed, but dashboard spacing changed by insertion. Manual viewport check is recommended.
